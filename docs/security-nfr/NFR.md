# Non-Functional Requirements (Security & Quality)

| ID | Название | Описание | Метрика/Порог | Проверка | Компонент | Приоритет |
|----|----------|-----------|---------------|----------|-----------|-----------|
| SEC-001 | Аутентификация API | Все API эндпоинты должны проверять аутентификацию | 100% запросов проходят проверку токена | API тесты, мониторинг | API Gateway | Высокий   |
| SEC-002 | Защита от брутфорса | Предотвращение подбора паролей и токенов | Макс 5 попыток в минуту с IP | WAF, rate limiting | Auth Service | Высокий   |
| SEC-003 | Валидация входных данных | Проверка всех входных параметров | 0 инцидентов SQL-инъекций | SAST, код-ревью | Все эндпоинты | Высокий   |
| PERF-001 | Время ответа API | Производительность критичных эндпоинтов | p95 < 500ms для /bookings | Load testing, monitoring | Booking Service | Средний   |
| PERF-002 | Доступность системы | Общая доступность сервиса | 99.5% uptime | Мониторинг uptime | Все компоненты | Средний   |
| REL-001 | Целостность бронирований | Гарантия отсутствия потерь данных | 0 потерянных броней при сбоях | Transaction logs, тесты | Database | Высокий   |
| REL-002 | Резервное копирование | Регулярное бэкапирование данных | Ежедневные бэкапы, RPO < 1 час | Backup reports | Database | Средний   |
| SCAL-001 | Масштабируемость | Обработка пиковых нагрузок | 100 RPS на эндпоинт /availability | Load testing | API, Database | Средний   |
| SEC-004 | Безопасность сессий | Защита сессионных данных | JWT TTL < 24 часа | Конфигурация | Auth Service | Средний   |
| SEC-005 | Шифрование данных | Защита PII и платежных данных | 100% HTTPS, шифрование в БД | SSL проверки | Network, DB | Высокий   |
