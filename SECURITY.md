# Security Policy

- Пожалуйста, **не** храните ключи/секреты в репозитории.
- Сообщайте об уязвимостях преподавателю/ТА через приватный канал.
- В коде используйте RFC 7807 problem-details с `correlation_id` (см. README).
- Во время курса используйте **синтетические** данные (без ПДн/платежей).

## Сообщение об уязвимостях

Безопасность нашего проекта крайне важна для нас. Если вы обнаружили потенциальную уязвимость безопасности, пожалуйста, сообщите об этом как можно скорее.

### Процесс сообщения об уязвимостях

1. **Немедленно отправьте email** на ivmushkat@edu.hse.ru с темой:
**Security Report: <краткое описание>**


2. **Включите в сообщение:**
- Подробное описание уязвимости
- Шаги для воспроизведения
- Возможное воздействие
- Предложения по исправлению (если есть)
- Ваши контактные данные для обратной связи

3. **Ожидайте ответа в течение 48 часов** с подтверждением получения отчета.


4. **Мы предоставим регулярные обновления** о прогрессе исправления каждые 3-5 дней.

### Секреты и инфраструктура
- Все секреты (включая `JWT_SECRET_KEY`) задаются через переменные окружения и не коммитятся в репозиторий.
- Запрещено логировать значения секретов; используйте `correlation_id` для трассировки инцидентов.
- Для локального тестирования сгенерируйте безопасный секрет: `export JWT_SECRET_KEY="$(python -c 'import secrets; print(secrets.token_hex(32))')"`.
